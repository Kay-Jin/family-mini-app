# 联调闭环（立刻做 · 约 15–30 分钟）

按顺序勾选；任意一步失败，看文末 **排错**。

## A. 云数据库（控制台）

1. 打开 **微信开发者工具** → 打开本项目 → **云开发** → **数据库**。
2. 确认或新建集合（可空集合）：
   - [ ] `households`
   - [ ] `household_members`
   - [ ] `members`
   - [ ] `invite_codes`
   - [ ] 以及 `README.md` 中列出的业务集合（`morning_briefs`、`check_ins` 等，可按需后补）
3. 开发阶段权限可先用 **仅创建者可读写** 或 **自定义只读 + 云函数写**（见 `docs/cloud-database-rules.md`）。

## B. 云函数 `family`

1. 在项目目录树找到 **`cloudfunctions/family`**。
2. 右键 → **上传并部署：云端安装依赖**（不是「增量」时最好选全量/云端安装依赖）。
3. 等待成功提示 → **云开发 → 云函数 → family** 看「更新时间」已变。
4. （可选）**云函数 → 日志**：随意进一次小程序 Tab，确认没有持续报错。

## C. 小程序端配置

1. 运行小程序 → **我的**。
2. **接口配置**：选 **CloudBase（微信云开发）**，填 **Env ID**，**保存**。
3. （建议）**清除工具缓存 / 本地存储** 或使用「新设备」模拟首次进入：`工具 → 清除缓存`（以你实际菜单为准）。

## D. 最小功能冒烟

### D1 新用户 / 无家庭

- [ ] 冷启动进入 **加入家庭**（`pages/onboarding`）。
- [ ] **创建家庭**：输入名称 → 进入 Tab **今日** 无报错。

### D2 邀请码（多人 + 上限）

- [ ] 账号 A：**家庭组织** → 设 **可用人次 ≥ 2** → 生成邀请码。
- [ ] 账号 B（另一微信或开发者工具切换调试）：**加入家庭** → 输入邀请码 → 成功。
- [ ] 账号 C：同一邀请码再加入 → 成功至 **`usedCount` 达 `maxUses`** 后应失败，提示「已达使用上限」或类似（以云函数 `message` 为准）。

### D3 多家庭（若只有你一个微信号）

- [ ] 同一用户用 **创建家庭** 再建第二个家庭（或用两账号互相邀请），使 **`household_members` 中该 openid ≥ 2 条**。
- [ ] 清掉本地 **`household_id` 存储**（或换设备）后冷启动：
  - [ ] 若服务端无法解析唯一活跃家庭 → 出现 **选择家庭** 列表。
- [ ] **我的 → 切换家庭**：列表出现且切换后 **今日** 数据随家庭变化（至少不报 `householdId is required`）。

### D4 退出家庭

- [ ] **我的 → 退出当前家庭**：
  - 若仍有其它家庭 → 留在小程序，不回到 onboarding；
  - 若已无家庭 → `reLaunch` 到 **加入家庭**。

---

## 排错

| 现象 | 建议 |
|------|------|
| `householdId is required` | 本地无 `household_id` 或云函数未部署新版本；重新部署 `family` 并确认已选 CloudBase Env。 |
| 成员列表空 | 查库 **`household_members`** 是否有多 `openid`+`householdId`；重进 **我的** 触发 `getOrCreateUser` 可迁移旧 `members`。 |
| 邀请码一直失败 | 看 **`invite_codes`** 文档是否已有 `usedCount`/`maxUses`；旧码仅有 `usedAt` 视为已消耗。 |
| 云函数无日志 | 确认小程序调用的是当前 **环境 ID**；真机需打开 **不校验合法域名** 等开发设置。 |

更完整清单见：`docs/deploy-checklist.md`。

# 家庭关怀小程序 UI/UX 设计系统（温馨风）

## 1. 目标与风格

- 设计目标：建立统一、可落地的完整 UI/UX 规范，服务当前四大主页面与 V1.1 扩展功能。
- 风格关键词：暖米色、木质感、可信赖、低压感、长辈友好。
- 适配对象：
  - 成人用户：信息密度适中、卡片分组清晰
  - 长辈用户：大字号、大按钮、低误触、强反馈

## 2. Design Foundations

### 2.1 色彩体系

| Token | 值 | 用途 |
|---|---:|---|
| `bgCanvas` | `#F7F1E8` | 全局画布背景 |
| `bgSubtle` | `#F3E9DC` | 二级区块背景 |
| `bgElevated` | `#FFFFFF` | 卡片背景 |
| `textPrimary` | `#2A221B` | 主文案 |
| `textSecondary` | `#655A4F` | 辅助文案 |
| `brand` | `#567A58` | 主操作色 |
| `brandPressed` | `#476849` | 按下态 |
| `accentWarm` | `#D6A35E` | 温暖强调色 |
| `accentWood` | `#8C6346` | 木质点缀 |
| `borderSubtle` | `rgba(42,34,27,0.10)` | 细边框 |
| `success` | `#3DBE6E` | 成功 |
| `attention` | `#E8B860` | 提醒 |
| `critical` | `#C44D4D` | 错误/风险 |

语义映射：
- 成功反馈：`success` + 简短正向文案
- 风险警示：`critical` + 可恢复动作
- 空状态：`textSecondary` + 引导按钮 `brand`

### 2.2 字体体系

| 角色 | 普通模式 | Senior 模式 |
|---|---:|---:|
| 大标题 `titleLg` | 36rpx | 42rpx |
| 标题 `titleMd` | 32rpx | 38rpx |
| 正文 `body` | 30rpx | 34rpx |
| 小正文 `bodySm` | 26rpx | 30rpx |
| 说明 `caption` | 24rpx | 28rpx |

字体策略：
- 中文优先系统字体（微信原生渲染稳定）
- 标题不超过两行，超长文案使用副文本承接

### 2.3 间距、圆角、阴影

- 8pt 网格：4/8/12/16/24/32 递进（见 `TOKENS.spacing`）
- 圆角：卡片 20rpx，输入框 14rpx，胶囊按钮 999rpx
- 阴影：
  - 卡片：`0 8rpx 20rpx rgba(86,122,88,0.10)`
  - 浮层：`0 10rpx 28rpx rgba(42,34,27,0.14)`

### 2.4 触控热区

- 普通模式最小点击高：88rpx
- Senior 模式最小点击高：96rpx
- 危险操作增加确认或防抖

## 3. 组件规范（含 Senior 变体）

### 3.1 Button

- 主按钮：`brand` 背景 + 白字 + 圆角 14rpx
- 文案长度：建议 <= 8 汉字；超长拆为“标题+说明”
- Senior：高度 96rpx，字号 34rpx

状态：
- 默认：`brand`
- 按下：`brandPressed`
- 禁用：背景 30% 透明 + 文案降级

### 3.2 Card

- 结构：标题区 / 内容区 / 操作区
- 卡片内间距 24rpx；卡片间距 20rpx
- 长列表避免嵌套多层卡片

Senior 变体：
- 标题更大，次要信息折叠
- 优先展示“下一步动作”

### 3.3 Input/Switch/Picker

- 输入框：浅边框、14rpx 圆角、16rpx 内边距
- 开关行：左文案右开关，保证同一行垂直居中
- 选择器：用明确“当前值 + 可变更”提示

Senior 变体：
- 行高增大
- 关闭复杂占位文案，强调结果导向

### 3.4 Bottom Tab

- 主导航固定 4 Tab：`今日/健康/回忆/我的`
- 图标风格统一线性/圆角，避免混用
- 当前项色彩增强，非当前项降噪

### 3.5 Feedback（Toast/Snackbar/错误态）

- 成功：短文案（<=10 字）
- 失败：给原因 + 下一步（如“重试”）
- 页面错误：卡片错误态替代白屏

## 4. 交互规范

### 4.1 页面状态

- `loading`：骨架或文案“加载中...”
- `empty`：说明当前无数据 + 操作引导
- `error`：错误卡片 + 可重试
- `offline`：提示网络不可用

### 4.2 关键路径约束

1. 报平安：1 次点击完成，成功后就地反馈并刷新列表
2. 上传相册：选择图片 -> 上传中 -> 成功回显
3. 可见性设置：修改即保存，失败回滚提示
4. 求助触发（V1.1）：2 步内完成，支持短时撤回

### 4.3 动效建议

- 页面切换：弱动效，避免眩晕
- 按钮反馈：按下态颜色变化即可
- Senior 模式减少复杂动效，保留必要反馈

## 5. 页面应用规范（Page Spec）

## 5.1 今日（`pages/today`）

- 信息优先级：晨报 -> 报平安 -> 最近记录
- 主 CTA：报平安按钮固定在首屏可见
- Senior：保留核心动作，弱化辅助信息

## 5.2 健康（`pages/health`）

- 卡片按时间倒序
- 步数与睡眠遵循可见性开关
- 无共享时展示友好占位语

## 5.3 回忆（`pages/memories`）

- 上传入口位于顶部卡片
- 列表卡片显示：缩略图、上传者、时间、标签
- Senior 或关闭共建时：只读提示替代上传按钮

## 5.4 我的（`pages/profile`）

- 分组顺序：账号与模式 -> 角色 -> 后端配置 -> 权限/隐私入口
- 配置项需“修改即有反馈”
- 避免在一屏中堆叠过多复杂表单

## 7. UI/UX 验收清单

### 7.1 视觉一致性

- [ ] 主背景、卡片、文案色遵循 token
- [ ] 四页标题层级一致
- [ ] 主按钮状态与圆角一致

### 7.2 可用性

- [ ] Senior 模式字号与热区显著提升
- [ ] 关键路径（报平安、上传）无多余步骤
- [ ] 错误态均可理解且可恢复

### 7.3 可实现性

- [ ] 样式均可通过小程序原生组件落地
- [ ] 不依赖重型自定义渲染框架
- [ ] token 可在代码中复用
